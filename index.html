卷积层

  • 和全连接层的对比和优点：
    • 假设是图片的话，全连接层是一维展开的，不能提取到图片的局部特征，而卷积层可以提取到局部特征。同时RGB通道的信息全连接层无法提取
    • 卷积层可以保持形状不变，当输入数据是图像时，卷积层会以3维数据的形式接收输入数据，并同样以3维数据的形式输出至下一层

langchain

发表于 2024-07-12 更新于 2024-07-30

概念梳理

chunk：想象肖安的词块，这里的chunk是类似于语言的基石一样的东东
stream:流式处理，个人理解就是产生多少内容就输出多少东西
astream：允许异步处理，但是这里面的细节我还没有熟悉
invoke：对模型的一次调用（触发）
chain=prompt|llm|output_parser 后面的parser是规定输出格式的,叫输出解析器

中文文档

  • 提示模块：负责将用户输入格式化为可以传递给语言模型的格式
  • 示例选择器
  • 聊天模型chatmodel：较新的语言模型形式，接收消息并输出消息
  • LLM：较旧的语言模型形式，接收字符串输入并输出字符串
  • 输出解析器：负责将LLM的输出解析为更结构化的格式
  • 文档加载器：负责从各种来源加载文档
  • 文本拆分器：将文档拆分为可用于检索的块
  • 嵌入模型：将一段文本转换为数值表示
  • 向量存储：有效存储和检索嵌入的数据库
  • 检索器：负责接收查询并返回相关文档（个人认为是这项任务的重头戏）
  • 索引：使向量存储与基础数据源保持同步的过程

聊天

  • 聊天模型缓存：langchain为聊天模型提供了一个可选的缓存层
    • 好处1：对于相同的结果可以减少对LLM提供程序的API调用次数节省费用
    • 好处2：减少对LLM提供程序的API调用次数来加快应用程序速度
  • 函数回调：工具调用
    • 对于工具的一些疑问：是当作函数触发吗？参数怎么办呢？

一个能调用函数的示例（终于好了太感动了呜呜呜(′д｀ )…彡…彡）

LCEL

Langchain表达式语言：一种声明式的方式，可以轻松地将链条组合在一起。

  • |符号：类似于unix管道操作符，它将不同的组件链接在一起，将一个组件的输出作为下一个组件的输入
  • prompt是一个BasePromptTemplate，接受输入并生成PromptValue
  • PromptValue传递给Value，model接收并生成字符串
  • 字符串被解析器将输入转换为字符串

如果想要添加上下文信息，可以这么写：

链条中多了setup_and_retrieval

Prompt（提示词）

  • 提示结构：
    • 指令：告诉模型该怎么做，如何使用外部信息，如何查询并构建out
    • 外部信息：充当模型的附加知识来源，可以手动插入到提示中，通过矢量数据库检索获得，或通过其他方式引入
    • 用户的查询
    • Out指示器（比如如果想让模型写代码，开头写一个import大概率它就会写代码了）

工具设计成能被模型调用的函数：它们的输入是被模型生成的，它们的输出会传送回去给模型

  • 一个工具由以下部分组成：
    • 名字
    • 这个工具是干什么的描述
    • 一个JSON参数定义工具的输入
    • 一个函数
    • 有两种方式触发工具调用：
      • 用正确的语句触发

创建工具的各种方法

对上面的一些特性总结： - 不止可以对工具有描述，工具里面用到的变量也可以有描述 - 返回的内容可以分为给模型的和其他用处的，用一个tuple整合（详见artifact） - 可以简单的直接定义函数，也可以通过改写subclass的方式定义工具，后者虽然麻烦但是实现的功能最多也最灵活，还有一个用Runnables定义的方式我跳过了（因为还不怎么了解Runnable）

怎么用聊天模型调用工具

  • 虽然这个名字看起来是聊天模型在调用工具，但是事实上不是这样的，模型生成了调用工具的语句，是否调用它取决于用户。如果说，你想要从一个没有结构的文本中extract output matching some schema，你可以给模型一个extraction tool进行提取，并用生成的结果作为最终的结果

NWDAF（Network Data Analytics Function）

SLA（Service Level Agreement，服务级别协议） 面对未来通信场景的复杂化，业务需求的多样化，业务体验的个性化推出 相关的技术目前未知，总结：一坨 * UE：用户终端 * SIP协议 * 一个应用层的信令控制协议，维护管道的，对交互管道进行控制的协议

  • SDP

  • 信令：通信设备之间传送的控制消息，从而对相关设备产生影响而引发下一步的动作

  • 信令要完成什么

    • AAA：Authentication，Authorization，Accounting：鉴权，授权，统计（统计流量）

  • 分类

    • 按功能分类（这一堆分类感觉没用？）
      • 线路信令：具有监视功能
      • 路由信令：具有选择功能
      • 管理信令：具有操作功能
    • 按信道分类
      • 共路信令：信令以统一格式的消息信令单元形式传送
      • 随路信令：信令在各自对应的语音信道中或固定分配的通道中传送
    • 按工作区域分
      • 用户线信令
      • 局间信令

  • 网元：由一个或多个机盘或机框组成，能够独立完成一定的传输功能

介绍？

  • 通信系统模型：信源——>发送设备——>信道——>接收设备——>信宿
  • 通过交换网络互连主机

网络协议：important

  • 协议规定交换的信息的格式，意义，顺序及针对信息或者事件所采取的动作
  • 三要素:语法，语义，时序

网络结构

  • 网络边缘：主机，网络应用
    • 客户/服务器应用模型：Web应用，文件传输FTP应用
    • P2P应用模型：通信在对等实体之间直接进行：QQ，Skype
  • 接入网络，物理介质：有线或无线通信链路
    • 接入网络干的事情：将网路边缘接入核心网
    • 用户在接入网络中关心：带宽，接入方式是共享/独占
    • 例子：都是利用不同频率传输，尽头分离得到信号
      • 数字用户线路：DSL，独占至中心局
      • 电缆网络：各家庭共享至电缆头端的接入网络
      • 无线接入网路
  • 网络核心：互联的路由器，网络之网络
    • 网络核心的关键功能：路由+转发
    • 路由：确定分组从源到目的传输路径

剪枝

目的:为了降低过拟合的风险
预剪枝:用剪枝后验证集的正确率和剪枝前的正确率比较,如果小就剪枝
后剪枝:先生成，然后选择叶子结点，如果剪掉叶子节点正确率增加则剪枝，否则不剪枝。
预剪枝 vs. 后剪枝
预剪枝:测试时间开销降低，训练时间开销降低，但是欠拟合风险增加
后剪枝:测试时间开销降低，训练时间开销增加，欠拟合风险基本不变。
泛化性:后剪枝通常优于预剪枝
